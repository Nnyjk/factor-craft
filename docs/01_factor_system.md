# 01. Factor 系统与节奏控制

## 1.1 基础模型

$$ F(t) = F_{base} + F_{tide}(t) + F_{player} + F_{random} $$

* `F_base`：维度基准值（主世界 50、下界 80、末地 20）。
* `F_tide`：潮汐项，采用复合正弦。
* `F_player`：玩家设施偏移。
* `F_random`：噪声项。

推荐离散更新：
$$ F_{t+1} = F_t + \Delta_{tide} + \Delta_{player} + \Delta_{random} - D(F_t - F_{base}) $$

## 1.2 慢节奏规则（关键）

* 潮汐周期建议：**96000~192000 tick（4~8 天）**。
* Factor 数值实时更新，但 **Tier 只在新的一天开始时判定**。
* 判定时机：`world_time % 24000 == 0`。
* 使用日均值 `F_dayAvg` + 趋势 `F_trend` + 迟滞参数 `H`。

$$ Tier_{day+1} = f(F_{dayAvg}, F_{trend}, H) $$

## 1.3 Tier 与日切结算

| 区间 | Tier | 说明 |
| :--- | :--- | :--- |
| <10 | 枯竭 | 安全维护窗口 |
| 10~30 | 低能 | 精密与省料窗口 |
| 30~70 | 稳定 | 标准运行窗口 |
| 70~90 | 高能 | 提效与高压窗口 |
| >90 | 过载 | 奇迹合成与灾害窗口 |

### 边界事件（按日结算）

* 30→70：活化风暴
* 70→90：裂隙预兆
* 90+ 持续：失稳共鸣
* 回落到稳定：回稳奖励

> 所有边界事件的触发资格可日内累计，但统一在新日广播和发奖。
